## Get the FLL Backend running locally

This guide walks you through cloning, building, and running the First Lego League Backend on your local machine.

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed before you begin:

    | Tool | Minimum version |
    |------|-----------------|
    | Java | 21              |
    | Maven | 3.9+           |
    | Git  | any recent      |

    Verify your Java version:

    ```bash
    java -version
    ```

    Verify your Maven version:

    ```bash
    mvn -version
    ```
  </Step>

  <Step title="Clone the repository">
    Clone the project from GitHub and change into the project directory:

    ```bash
    git clone https://github.com/UdL-EPS-SoftArch-Igualada/first-lego-league-backend.git
    cd first-lego-league-backend
    ```
  </Step>

  <Step title="Build the project">
    Compile the source code and run Checkstyle validation:

    ```bash
    mvn package
    ```

    This command compiles the code, runs Checkstyle, and packages the application into a JAR file under `target/`.

    <Tip>
      To skip Checkstyle during a quick build, add `-Dcheckstyle.skip=true` to the Maven command.
    </Tip>
  </Step>

  <Step title="Run the application">
    Start the Spring Boot server:

    ```bash
    mvn spring-boot:run
    ```

    The server starts on port **8080** using the default (development) profile, which uses an in-memory H2 database.

    On startup, the application creates a default user:

    | Username | Password   | Email              |
    |----------|------------|--------------------|
    | `demo`   | `password` | demo@sample.app    |

    <Note>
      The default profile uses H2 as an in-memory database. The **production profile** (`-Dspring.profiles.active=prod`) targets PostgreSQL and tightens logging to `error` level only. Set the `DATABASE_URL` environment variable accordingly when running with `prod`. See [Running locally](/guides/running-locally) for full details.
    </Note>
  </Step>

  <Step title="Explore the API with HAL Explorer">
    Open your browser and navigate to:

    ```
    http://localhost:8080/explorer
    ```

    HAL Explorer lets you browse and interact with all hypermedia-driven REST endpoints without any additional tooling.

    <Warning>
      The H2 web console is **disabled by default** (`spring.h2.console.enabled: false`). Do not enable it in production.
    </Warning>
  </Step>

  <Step title="Run the tests">
    Execute the full test suite (unit tests and Cucumber acceptance tests):

    ```bash
    mvn test
    ```

    Test results are printed to the console. Cucumber reports are also published to the configured Cucumber Reports collection.
  </Step>
</Steps>
